# Claude Code Skill of the Day: Hooks

## What are Hooks?

Hooks in Claude Code let you run shell commands automatically at specific points
in Claude's lifecycle — for example, before a tool runs, after it completes, or
when a session starts. They act like event listeners that give you programmatic
control over Claude's behavior without touching your prompts. This is perfect for
things like auto-formatting code after edits, logging activity, or running tests
whenever a file changes.

## Copy-Pasteable Example

Add this to your `.claude/settings.json` to auto-lint after every file edit:

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "npm run lint --fix"
          }
        ]
      }
    ]
  }
}
```

This fires `npm run lint --fix` automatically every time Claude edits or writes a
file — no need to ask Claude to lint, it just happens.

## Available Hook Events

| Event | When it fires |
|---|---|
| `PreToolUse` | Before a tool executes (can block it) |
| `PostToolUse` | After a tool completes |
| `Notification` | When Claude sends a notification |
| `Stop` | When Claude finishes its turn |
| `SessionStart` | Once when a new session begins |

## Try It Yourself

Add a simple logging hook to see how arguments are passed:

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit",
        "hooks": [
          {
            "type": "command",
            "command": "echo \"Claude edited a file at $(date)\" >> /tmp/claude-edits.log"
          }
        ]
      }
    ]
  }
}
```

Then watch `/tmp/claude-edits.log` grow as Claude makes edits in your session!
