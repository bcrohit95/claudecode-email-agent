# Claude Code Skill of the Day: Hooks

## What are Hooks?

Hooks let you run shell commands automatically at specific points in Claude Code's lifecycle — before a tool runs, after it completes, when a session starts, or when it stops. They're defined in your settings.json and execute outside of Claude's awareness, giving you deterministic control over the environment without relying on Claude to remember to do it.

## Copy-Pasteable Example

Add this to your `~/.claude/settings.json` to automatically run your linter after every file edit:

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "cd $CLAUDE_PROJECT_DIR && npx eslint --fix $CLAUDE_TOOL_OUTPUT_FILE_PATH 2>/dev/null || true"
          }
        ]
      }
    ]
  }
}
```

Other available hook events:
- `PreToolUse`  — runs before a tool executes
- `PostToolUse` — runs after a tool executes
- `SessionStart` — runs when a new Claude Code session begins
- `SessionStop`  — runs when a session ends

## Try It Yourself

Open your settings with `/config`, switch to the JSON view, and add a `SessionStart` hook that logs the time:

```json
{
  "hooks": {
    "SessionStart": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "echo 'Claude Code session started at $(date)' >> ~/claude-sessions.log"
          }
        ]
      }
    ]
  }
}
```

Start a new Claude Code session and check `~/claude-sessions.log` — you should see the timestamp logged automatically.
