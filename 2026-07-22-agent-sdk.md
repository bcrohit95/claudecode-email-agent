# Claude Code Skill: Agent SDK

The Claude Agent SDK lets you build your own automated agents that run Claude programmatically — you define the tools, the loop, and the instructions, and the SDK handles message passing, tool execution, and retries. This is the foundation for scheduled tasks, multi-step pipelines, and anything that needs Claude to act without a human typing at a terminal. Unlike the raw Messages API, the Agent SDK manages the back-and-forth agentic loop for you so you focus on *what* the agent does, not *how* it iterates.

## Copy-Pasteable Example

```python
import anthropic

client = anthropic.Anthropic()

tools = [
    {
        "name": "read_file",
        "description": "Read a file from disk and return its contents.",
        "input_schema": {
            "type": "object",
            "properties": {
                "path": {"type": "string", "description": "Absolute path to the file"}
            },
            "required": ["path"]
        }
    }
]

def handle_tool(name, input):
    if name == "read_file":
        with open(input["path"]) as f:
            return f.read()
    return "unknown tool"

messages = [{"role": "user", "content": "Read /etc/hostname and tell me the machine name."}]

# Agentic loop — runs until the model stops calling tools
with client.beta.messages.tool_runner(
    model="claude-sonnet-5",
    max_tokens=1024,
    tools=tools,
    messages=messages,
) as runner:
    for event in runner:
        if event.type == "tool_use":
            result = handle_tool(event.name, event.input)
            runner.submit_tool_result(event.id, result)
    print(runner.final_message.content[0].text)
```

## Try It Yourself

Install the SDK (`pip install anthropic`), swap in a real file path, and run the snippet above. Then add a second tool — say, `list_dir` — and ask the model to explore a directory. Watch how the SDK automatically continues the loop each time Claude calls a tool, without you writing any retry or loop logic.
