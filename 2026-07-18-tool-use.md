# Tool Use

Tool Use lets you give Claude a set of functions (tools) it can call during a conversation to take real actions — like searching the web, reading files, running code, or querying a database. Instead of just generating text, Claude decides when and how to call your tools, interprets the results, and continues reasoning until it has a final answer.

## Example

```python
import anthropic

client = anthropic.Anthropic()

tools = [
    {
        "name": "get_weather",
        "description": "Get the current weather for a city.",
        "input_schema": {
            "type": "object",
            "properties": {
                "city": {"type": "string", "description": "City name"}
            },
            "required": ["city"]
        }
    }
]

response = client.messages.create(
    model="claude-sonnet-5",
    max_tokens=1024,
    tools=tools,
    messages=[{"role": "user", "content": "What's the weather in Tokyo?"}]
)

# Check if Claude wants to use a tool
for block in response.content:
    if block.type == "tool_use":
        print(f"Claude called: {block.name}")
        print(f"With input: {block.input}")
        # --> You'd call your real get_weather() here, then send the result back
```

## Try it yourself

Add a simple `calculator` tool with fields `operation` (add/subtract/multiply/divide) and two numbers. Ask Claude "What is 137 times 49?" and watch it invoke the tool instead of computing in its head — then feed the result back and see it form a natural language answer.
