# Agent SDK

## What It Is
The Claude Agent SDK lets you build custom agents that run inside Claude Code — each agent is a specialized process with its own tools, instructions, and context window. You wire agents together so Claude Code can delegate subtasks (like code review, deployment checks, or data lookup) to the right specialist automatically. Think of it as building a team of focused AI helpers that Claude Code can call on demand.

## Copy-Pasteable Example

```python
# agents/code-reviewer.md  (place in your .claude/agents/ directory)
---
name: code-reviewer
description: Reviews diffs for bugs, security issues, and style problems. Use when the user asks for a code review or before merging a PR.
tools: Read, Bash, WebSearch
---
You are a senior engineer performing a focused code review.
Review only for correctness bugs, security vulnerabilities, and clear logic errors.
Be concise: list findings as bullet points with file:line references.
Skip style nit-picks unless they affect correctness.
```

Once saved, Claude Code automatically discovers and routes to this agent when relevant.
You can also invoke it explicitly in chat:

```
/agent code-reviewer review the diff on this branch
```

## Try It Yourself
Create a `.claude/agents/` directory in your project, add a Markdown file like the example above, and start a new Claude Code session. Ask Claude to review your latest changes — it will automatically delegate to your custom agent. Try giving your agent a narrow set of tools (e.g., only `Read` and `Bash`) to keep it focused and fast.
